页面的置换算法(Page Replacement Algorithms)

缺页: 需要的页没有装入物理内存中。

由于程序的局部性，操作系统才能实现页面的置换。

当空闲空间大小达到一个阈值时，会触发淘汰机制Page Stealer。

我想扔页面，该怎么扔？(ﾟ∀。)7

  • OPT - 最优页面置换算法(Optimal page replacement algorithm)效率最高的，但是无法实现

    扔掉以后再也用不到的。事个理论算法，可以作为一个标杆，用来事后分析。

    会出计算题

  • 随便扔据说还不错

  • NRU - 最近未使用(Not recently used page replacement)挺不咋地的，但是现代操作系统大量使用，简单粗暴

    R位为0，M位为0的最先扔，再扔R位为0，M位为1的。

  • FIFO - 先进先出(First-In, First-Out page replacement)最拉胯的算法

    会出计算题

  • SC - 第二次机会(Second chance algorithm)对FIFO的重要改进

    检测先来的进程，如果R位为0，即刻丢掉；如果R位为1，将其R位清零，放在队尾。

  • 时钟算法(Clock page replacement)

    和 S C 完 全 一 致。

  • LRU - 最近最少使用(Least recently used page replacement)这个事好的

    会出计算题

    • 老化算法(Aging)LRU的一个近似实现

      每个页面配一个8位的记录值，每个时钟滴答的使用状态记录在最高位(记录前整体右移)，状态会保留8个时钟滴答。

  • 工作集置换算法(Working set page replacement)

    R位为1的保留，同时把该页面年龄改为当前时间，R位为0的如果页面年龄大于$$的扔掉。

    • 工作集时钟置换算法(WSClock page replacement)

      对工作集置换算法的的改进。

百拉迪异常(Belady anormaly)

在某种情况下，给进程分会配更多的页帧，缺页情况可能反而更多。

只有FIFO会发生百拉迪异常。
