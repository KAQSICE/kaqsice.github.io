死锁(Deadlocks)

什么事死锁

A set of process is deadlocked if each process in the set is waiting for an event that only another process in the set can cause.

死锁的四个条件

  1. 互斥(Mutual exclusion condition)
  2. 部分拥有，期待更多(Hold and wait condition)
  3. 不可抢夺(No preemption condition)
  4. 循环等待(Circular wait condition)

对待死锁的态度和方法

  1. 就当他不存在(Just ignore the problem)鸵鸟算法——不二之选

  2. 发现和恢复(Detection and recovery. Let deadlocks occur, detect them, take action)

    沙了最年轻的进程罢(´_ゝ`)

  3. 动态避免(Dynamic avoidance by careful resource allocation)

  4. 主动预防(Prevention, by structurally negating one of the four required conditions)

银行家算法划重点

不要让系统进入非安全状态。

具体算法请见课本。

死锁的预防

  • 破坏互斥(Attacking the mutual exclusion condition)
    • 假脱机(Spooling)
  • 要么不给，要么都给(Attacking the hold and wait condition)
  • 抢夺(Attacking the no preemption condition)不如把它沙了(`ε´ )
  • 有序分配(Attacking the circular wait condition)
