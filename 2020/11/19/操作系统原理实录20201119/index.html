操作系统原理实录20201119

存储管理(Storage Management)

什么事文件?

基本上放在外存上的东西都是文件数据库除外

文件的定义:

A collection of data.

A stream of characters or a “byte stream”

文件可以按名存取

文件的类型:

  • 普通文件(Ordinary file)
  • 目录文件(Directory file)
  • 特殊文件(Special file)如设备

文件的命名

*NIX系统文件名大小写敏感

Windows系统大小写不敏感

文件的操作

  • 创建(create)
  • 删除(delete)
  • 打开(open)
  • 关闭(close)
  • 读取(read)
  • 写入(write)
  • 增加(append)
  • 查找(seek)
  • 获取标签(get attributes)
  • 设置标签(set attributes)
  • 重命名(rename)

Inodes

在UNIX系统里，文件可以分为三部分

  1. 文件的名称
  2. 文件的属性
  3. 文件的内容

文件的属性通过IndexNode存储，即Inode，每个文件只有一个，不存放文件的名字。

文件的名字存在其所在的目录里。目录的内容就是目录内文件的名字和文件的Inode。

一个文件是可以有多个名字的，我们把名字叫做Link。

Inode内会记录一个Link Counter，可以得到该文件的Link数。

共享文件(Shared Files)

ln命令使用

$ls -li

总用量 0
1054885 -rw-r–r-- 1 tranced tranced 0 11月 19 14:12 source

$ln source target

$ls -li

总用量 0
1054885 -rw-r–r-- 2 tranced tranced 0 11月 19 14:12 source
1054885 -rw-r–r-- 2 tranced tranced 0 11月 19 14:12 target

$ls -li

总用量 0
1054922 -rw-r–r-- 1 tranced tranced 0 11月 19 14:15 source

$ln -s source target

$ls-li

总用量 4
1054922 -rw-r–r-- 1 tranced tranced 0 11月 19 14:15 source
1054961 lrwxrwxrwx 1 tranced tranced 6 11月 19 14:19 target -> source

存储方式

顺序存储

适合随机存取

不方便增删

容易产生碎片(外碎片)

链式存储

方便增删

不适合随机存取

既然顺序存储和链式存储都很拉胯，那就使用树形结构罢！

Linux中Inode大小为128字节，68字节存放文件标签，60字节存放指针，共15个指针。

12个指针直接指向数据块(设数据块大小1KBext4基本上是4KB，最大可以存放约4TB的单文件)，第13个指针指向一个数据块，数据块中事256个指针，指向256个数据块(256KB)。第14个指针也指向一个数据块，数据块中256个指针每个都指向一个256指针的数据块，即2562(64MB)。第15个同理(16GB)。
