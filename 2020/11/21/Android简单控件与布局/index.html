Android简单控件与布局

关于Layout

什么是Layout

Layout，翻译成中文即布局，Layout决定控件在界面上如何呈现(包括位置，大小，动画等逻辑)。对于Layout，官方给出了如下介绍

布局可定义应用中的界面结构（例如Activity的界面结构）。布局中的所有元素均使用 View 和 ViewGroup 对象的层次结构进行构建。View 通常绘制用户可查看并进行交互的内容。然而，ViewGroup 是不可见容器，用于定义 View 和其他 ViewGroup 对象的布局结构，如图所示。

View 对象通常称为“微件”，可以是众多子类之一，例如 Button 或 TextView。ViewGroup 对象通常称为“布局”，可以是提供其他布局结构的众多类型之一，例如 LinearLayout 或 ConstraintLayout。

我要怎么声明布局呢?

在Android开发中，声明布局有两种方法:

  • 使用XML布局文件声明布局

关于XML

啥是XML? 点这里看维基百科。

可扩展标记语言（英语：Extensible Markup Language，简称：XML）是一种标记语言。标记指计算机所能理解的信息符号，通过此种标记，计算机之间可以处理包含各种信息的文章等。如何定义这些标记，既可以选择国际通用的标记语言，比如HTML，也可以使用像XML这样由相关人士自由决定的标记语言，这就是语言的可扩展性。XML是从标准通用标记语言（SGML）中简化修改出来的。

XML严格区分大小写，有严格的树状结构。在一个Layout文件中，只能有一个根元素，且必须是View对象或者ViewGroup对象。在编译时，每个XML文件都会被编译成View资源。在Activity的onCreate()回调实现内通过调用setContentView()方法来加载布局(如下)。

来写一个布局吧

新建一个布局文件

Android原生应用的资源文件存放在res目录，下面会有很多子目录，不同的目录存放不同类型的资源文件，常用如下:

子目录 用途
drawable/ 存放位图文件（.png、.9.png、.jpg、.gif）或编译为以下可绘制对象资源子类型的 XML 文件
mipmap/ 存放启动器图标
layout/ 存放布局文件
values/ 包含字符串、整型数和颜色等简单值的 XML 文件

这里要新建一个布局文件，所以在layout/目录下新建一个XML文件即可。

XML中的属性

XML的元素中可以包含各种属性，在Android开发中常用的有id、layout_width、layout_height等等。

不同的控件有不同的特有属性，如ImageView中的src属性(ImageView是用来显示图片的控件，src指定图像文件的路径)

几种常用的布局

约束布局(ConstraintLayout)永远滴神

约束布局使用约束的方式来指定各个控件的位置关系。

常用的相对位置属性:

layout_constraintLeft_toLeftOf
layout_constraintLeft_toRightOf
layout_constraintRight_toLeftOf
layout_constraintRight_toRightOf
layout_constraintTop_toTopOf
layout_constraintTop_toBottomOf
layout_constraintBottom_toTopOf
layout_constraintBottom_toBottomOf
layout_constraintBaseline_toBaselineOf
layout_constraintStart_toEndOf
layout_constraintStart_toStartOf
layout_constraintEnd_toStartOf
layout_constraintEnd_toEndOf

线性布局(LinearLayout)

线性布局是按照水平或垂直的顺序将子元素(可以是控件或布局)依次按照顺序排列，每一个元素都位于前面一个元素之后。

线性布局分为两种: 水平方向和垂直方向的布局。分别通过属性android:orientation="vertical"和android:orientation="horizontal"来设置。

android:layout_weight表示子元素占据的空间大小的比例。

相对布局(RelativeLayout)

RelativeLayout是其按照子元素之间的位置关系完成布局的，非常适合于一些比较复杂的界面设计(ConstraintLayout似乎要更加强大)。

常用的相对位置属性:

layout_toLeftOf
layout_toRightOf
layout_above
layout_below
layout_alignParentLeft
layout_alignParentRight
layout_alignParentTop
layout_alignParentBottom
layout_centerInParent
layout_centerHorizontal
layout_centerVertical

常用的控件

显示文字，就用TextView罢

TextView很显然是用来呈现text的控件，主要会用到的属性就是和text相关的属性:

属性 用途
text 用来指定显示的文字
textColor 显示文字的颜色
textSize 显示文字的尺寸(最好使用sp为单位，但不要用作布局尺寸，关于dp和sp的区别，戳这里)

当然还有大把别的属性，就不多谈了。

显示图片，就用ImageView罢

ImageView用来显示图片，用的比较多的是src属性，用来指定图片的位置。不过原生的ImageView是不支持网图和动图的，所以在加载图片时一般使用第三方框架进行加载(如Glide、PicassoPS: 这货似乎不能加载GIF)

显示按钮，就用Button罢

Button就是很普通的按钮，可以像TextView一样直接设置文字。

其它常用控件

其它还有很多控件如CardView用来显示卡片，EditText用来输入文字等等。

也有一些使用起来比较复杂的，比如一些需要Adapter才能使用的控件，如显示列表的ListView、更强大的列表RecyclerView以及网格视图GridView等等。

这些就不详细说明了，想用的时候Google一下用法即可，常用的RecyclerView后面也会做讲解。
